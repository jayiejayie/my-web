# 如何开展安全风险分级管控

## ——基于蝴蝶结模型（Bow-Tie）的工程实践分享

---


## 第一章 引言

### 1.1 为什么写这篇文章

2021年修订的《安全生产法》将"安全风险分级管控和隐患排查治理双重预防机制"写入法律条文，这意味着双控机制已经从一个"可选的先进做法"变成了企业的法定责任。

但回到工程现场，一个现实的问题始终摆在每个项目面前：**这到底怎么干？**

很多项目接到文件要求后，做法是让安全员加班赶出一套"体系文件"，填几张风险评价表，挂几块风险公示牌。至于这些表格和现场实际施工有多大关系、风险辨识有没有遗漏、管控措施到底能不能落地——往往没有经过认真推敲。

这正是本文要解决的核心问题。

我们不重复罗列文件条款，而是从**工程管理人员的视角**出发，围绕一个完整的施工场景——基础工程土方开挖——从头到尾走一遍风险分级管控的实施流程。在这个流程中，我们引入了一个在国际上已经广泛应用、但在国内建筑行业还不太常见的工具：**蝴蝶结模型（Bow-Tie）**。

### 1.2 政策演进的四个阶段

回看过去近十年，双控机制的建设经历了一个从"理念提出"到"顶层设计"再到"全面立法"的清晰轨迹。

**第一阶段：理念萌芽（2015-2016年）**

2015年12月，习近平总书记在主持中央政治局常委会会议时首次提出："对易发重特大事故的行业领域采取风险分级管控、隐患排查治理双重预防性工作机制，推动安全生产关口前移。"这是"双重预防机制"概念的首次提出。随后，2016年1月全国安全生产电视电话会议明确要求在高危行业领域推行这一机制。

**第二阶段：顶层设计（2016年）**

2016年4月28日，国务院安委办印发《标本兼治遏制重特大事故工作指南》（安委办〔2016〕3号），首次在国家文件中系统阐述了双重预防机制的建设目标——"到2018年，构建形成点、线、面有机结合、无缝对接的安全风险分级管控和隐患排查治理双重预防性工作体系"。

2016年10月9日，国务院安委办印发《关于实施遏制重特大事故工作指南构建双重预防机制的意见》（安委办〔2016〕11号），这是双控机制的纲领性文件，明确提出了"把风险控制在隐患形成之前、把隐患消灭在事故前面"的核心理念，并对企业如何构建双重预防机制作出具体部署。

2016年12月9日，中共中央、国务院印发《关于推进安全生产领域改革发展的意见》，将"构建风险分级管控和隐患排查治理双重预防工作机制"写入中央文件，上升为安全生产领域改革的重要任务。

**第三阶段：地方实践与法律确立（2017-2021年）**

2017年至2019年间，各地陆续出台实施细则。河南省2019年2月印发《房屋建筑和市政工程施工安全风险隐患双重预防体系建设实施细则（试行）》，同年黑龙江省发布《建筑施工安全风险分级管控和隐患排查治理双重预防机制》（黑建规范〔2019〕1号）。住建部2018年发布《危险性较大的分部分项工程安全管理规定》（住建部令第37号），2019年发布《工程质量安全手册（试行）》，为双控机制在建筑施工领域的落地提供了配套支撑。

2021年6月10日，第十三届全国人大常委会第二十九次会议通过《关于修改〈中华人民共和国安全生产法〉的决定》，将双重预防机制正式写入法律。新《安全生产法》第四条明确要求生产经营单位"构建安全风险分级管控和隐患排查治理双重预防机制"，第二十一条将"组织建立并落实双重预防工作机制"列为主要负责人的法定职责，第四十一条进一步规定了具体工作要求。双控机制从政策倡导上升为法律义务。

**第四阶段：标准完善与深化推进（2022年至今）**

2022年4月19日，住建部印发《房屋市政工程生产安全重大事故隐患判定标准（2022版）》（建质规〔2022〕2号），为重大隐患的认定提供了明确标尺。同年3月，住建部开展房屋市政工程安全生产治理行动，将双重预防机制建设作为重点任务之一。

2024年10月，广东省住建厅发布《关于加强房屋市政工程施工安全风险分级管控和隐患排查治理双重预防机制的通知》，进一步明确了建设单位首要责任等监管要求。2025年3月，天津市住建委正式印发《房屋建筑和市政基础设施工程施工安全风险分级管控和隐患排查治理双重预防机制建设实施指南》，这是目前建筑行业最系统的一份省级操作指南，标志着双控机制建设进入了标准化、精细化的新阶段。

回顾这条脉络，结论很清楚：双控不是一阵风，而是安全管理体系的基础架构。

### 1.3 传统做法的三个痛点

在与多个项目的交流中，我们发现常规的风险分级管控做法普遍存在三个问题。

**痛点一：辨识靠经验，容易有盲区。**

大多数项目的风险辨识依赖安全员的个人经验，或者照搬其他项目的清单。一个直接后果是：某个风险是不是真的被覆盖到了，没有人能给出一个"系统性的保证"。缺少一个框架来检查"我们有没有漏掉什么"。

**痛点二：措施描述笼统，缺少层级感。**

"加强管理""严格执行操作规程""做好安全防护"——这些表述在风险清单中非常常见。但问题在于：一个笼统的措施，无法指导现场行动。某个高风险路径上，是应该有工程物理措施还是管理措施？是否有纵深防御？这些往往没有明确的设计逻辑。

**痛点三：风险管控清单和隐患排查"两张皮"。**

风险管控清单和安全检查表往往是两套独立的东西。风险管控清单做完之后锁在文件柜里，日常排查还是用传统的安全检查表。文件和实际两张皮，最终双控变成了"应付检查"的表面文章。

这三个痛点，指向同一个症结：**缺少一个能把风险"可视化、结构化、可验证"的分析工具。**

### 1.4 本文的结构

第二章先建立核心概念：双控的因果逻辑是什么、Bow-Tie的基本结构长什么样。

第三章是全文的主体，沿着"组织机构→确定风险点→危险源辨识→风险评价和分级→制定管控措施→重大风险评估→风险告知和分级管控"这七个步骤，逐层展开。每一步都给出方法说明、Bow-Tie对应的核心概念，以及土方工程案例的实操演示。

第四章到第六章分别讨论各方职责、持续改进机制和总结建议。

## 第二章 核心概念与逻辑框架

在进入具体操作之前，有必要先把几个基本概念理清楚。这些概念是后续所有分析的基础。

### 2.1 一条因果链和两道防线

双重预防机制的本质，可以用一条因果链和两道防线来概括。

**因果链：**

```
危险源 → 风险 → 管控措施失效 → 隐患 → 不治理 → 事故
```

分析一下每个节点的含义：
- **危险源**：能量的载体。基坑边坡土体的重力势能、挖掘机的动能、配电箱的电能——危险源本身不构成问题，它是工程中客观存在的。
- **风险**：危险源失控的可能性和后果的组合。风险是抽象的，是一个"可能性"的估计。
- **管控措施失效**：当本应控制住危险源的措施不起作用了，风险就变成了隐患。
- **隐患**：管控措施失效后的状态。隐患可以观测、可以排查。
- **事故**：隐患没有得到治理，最终导致能量意外释放，造成人员伤亡或财产损失。

因果链告诉我们的核心道理是：**事故不是一个孤立的事件，而是一条失效链条的终点。** 越早干预，成本越低，效果越好。

基于这个逻辑，双控机制架设了两道防线：

```
第一道防线：安全风险分级管控
  干预位置：危险源 → 隐患（关口前移，事前预防）
  核心任务：辨识危险源、评估风险、设计措施、把风险控制在可接受水平

第二道防线：隐患排查治理
  干预位置：隐患 → 事故（事中控制）
  核心任务：排查隐患、整改闭合、防止隐患演变为事故
```

国务院安委办〔2016〕11号文件用一句话说透了这两道防线的定位：**"把风险控制在隐患形成之前，把隐患消灭在事故前面。"**

本文聚焦第一道防线——风险分级管控。至于第二道防线，我们会在第三章第七步中讨论它与第一道防线的联动关系。

### 2.2 风险分级管控的七个步骤

综合对比国务院安委办〔2016〕11号文件、住建部《工程质量安全手册（试行）》、天津市《实施指南》（2025）、河南省《实施细则》（2019）、黑龙江省《双重预防机制》（2019）、广东省《通知》（2024）等十余份权威文件，风险分级管控的实施流程可以归纳为**七个核心环节**：

```
第一步：组织机构搭建与制度建设
第二步：确定风险点
第三步：危险源辨识
第四步：风险评价与分级
第五步：制定管控措施
第六步：重大及较大风险控制措施评估
第七步：风险告知与分级管控
```

**各省表述略有差异，但核心逻辑一致**：

- **国务院安委办〔2016〕11号**（纲领性文件）：提出"全面开展安全风险辨识、科学评定安全风险等级、有效管控安全风险、实施安全风险公告警示"四步框架
- **住建部《工程质量安全手册》**（2018）：强调"风险识别→风险评估→风险控制→风险监测"的闭环管理
- **天津市**（2025）：七步流程最完整，首次明确将"重大及较大风险控制措施评估"作为独立步骤
- **河南省**（2019）：最早将"两张清单"（风险管控清单、隐患排查清单）作为核心工具
- **黑龙江省**（2019）：强调"班前风险确认"制度，将风险告知前置到每日作业前
- **广东省**（2024）：突出建设单位首要责任，要求"一风险一档案"，重大风险动态清零

其中，第二步到第六步是一个典型的**风险分析循环**（识别→评估→控制→验证），第一步和第七步分别是**管理基础**（谁来干）和**管理闭环**（如何持续）。本文的第三章将按照这七个步骤逐层展开。

### 2.3 理解蝴蝶结模型（Bow-Tie）的基本结构

Bow-Tie模型以**初始事件**为中心，左右对称展开。

```
危险源 ──→ 威胁 ──→ 控制措施 ──→ 初始事件 ──→ 应急措施 ──→  后果
                        │                       │
                        ▼                       ▼
                  关键保障活动                  关键保障活动
```

这个模型包含七个要素，可以用一个土方工程的实例来理解：

| 要素 | 本质 | 土方工程中的对应 |
|:----|:----|:----------------|
| **危险源** | 能量的载体 | 基坑边坡土体（具有重力势能） |
| **威胁** | 直接触发因素 | 坡度过陡、堆载超限、降雨侵蚀 |
| **初始事件** | 危险源失控的中性状态 | 基坑边坡坍塌 |
| **后果** | 最终损失 | 人员被埋伤亡 |
| **控制措施** | 阻止威胁释放危险源能量 | 放坡、限载、排水沟 |
| **应急措施** | 减轻后果损失 | 应急通道、应急物资 |
| **关键保障活动** | 验证措施是否持续有效 | 坡度检查、物资巡检 |

注意**初始事件**这个概念——它是Bow-Tie模型的中心分水岭。它描述的是危险源"失控"这个中性状态，而不是最终的灾难后果。以基坑边坡土体为例：

- "基坑边坡坍塌"是初始事件（中性、客观描述）
- "人员被埋伤亡"是后果（灾难结果）
- "坡度过陡"是威胁（触发原因）

这个区分很重要。在实际分析中，把"初始事件"和"后果"混为一谈，是导致风险辨识混乱的最常见原因之一。

### 2.4 为什么在双控中引入Bow-Tie

传统的风险分级管控（比如JHA法），是"单线"思维——找出危险源，判断可能性和后果严重性，算出风险值，然后提出几句"加强管理"之类的措施。

Bow-Tie把这种"单线"变成了"链式"思维：
- 一个危险源可能有多个威胁——每个威胁都是一个不同的失效路径
- 每个威胁应该有多个不同层级的控制措施——阻止它触发初始事件
- 每个措施要有对应的保障活动——确保它持续有效

换句话说，传统方法要求你"列出一张风险清单"，Bow-Tie要求你"把每个风险的因果链想透"。这两种思维的区别，在后面的案例分析中会越来越清楚。

### 2.5 一个更深层的视角

如果把风险管控拆解得足够细，会发现它本质上在做同一件事：**在动态变化中维持平衡。** 危险源释放的能量可以看作一种"主动的、外显的力量"——它具有破坏倾向；而约束能量的管控措施则是"收敛的、制约的力量"——它负责保持稳定。安全不是消灭了前者，而是让后者始终跟得上前者的变化。风险失控的本质，就是这种平衡被打破了：要么是能量超出了预期，要么是约束失效了。

这个视角可以帮助我们理解为什么"写制度"往往不够——制度是静态的约束，而风险是动态的。好的管控体系不是一座静止的堡垒，而是一套能在变化中自动调校平衡的机制。这正是后面七步流程要解决的问题。

## 第三章 风险分级管控——七步实施流程

本章是全文的核心。我们以**基础工程-土方工程**为贯穿案例，按照前述七个步骤，逐层展示风险分级管控的完整实施过程。在每个步骤中，同步融入Bow-Tie的对应概念和方法。


---

### 3.1 第一步：组织机构搭建与制度建设

在开始任何技术分析之前，必须先解决一个问题：**谁来组织这件事？按什么规则做？**

这一步是整个双重预防机制能够运转的组织基础。如果组织架构没搭好、制度没建全，后面再好的分析方法也难以落地。

#### 3.1.1 组织架构

双控机制的建设不能交给安全部门一个部门去完成，它需要生产、技术、设备、物资等各条线的共同参与。因此，建立一个跨部门的领导小组是第一步。

**企业层面**，应由主要负责人担任组长，分管生产、安全、技术的负责人担任副组长。领导小组下设办公室——通常设在安全管理部门——由各职能部门负责人作为成员。这种架构的目的是确保双控建设不是一个"安全部的事"，而是整个企业的共同任务。

**项目层面**，应由项目经理担任组长，项目安全总监和技术负责人担任副组长。成员应包括项目各岗位管理人员及各分包单位负责人。项目层面尤其需要明确一点：**班组长是一线落实的关键角色**，必须纳入工作小组。

#### 3.1.2 五项基本制度

国务院安委办〔2016〕11号文件要求企业建立健全安全风险分级管控和隐患排查治理制度，天津市、山东省等省级指南进一步细化为五类基本制度：

| 制度 | 核心内容 | 主要依据 |
|:-----|:---------|:---------|
| 风险分级管控制度 | 工作职责、风险点确定、危险源辨识、风险评价、措施制定与评估、风险告知、文件管理、持续改进 | 国务院安委办、各省指南 |
| 隐患排查治理制度 | 隐患分级、排查清单、排查计划、治理程序、验收销号 | 国务院安委办、住建部 |
| 教育培训制度 | 培训计划、对象与内容、考核标准 | 新《安全生产法》、各省指南 |
| 考核奖惩制度 | 考核指标、频次、奖惩措施 | 天津市、山东省指南 |
| 文件管理制度 | 资料分类、建档、归档、查阅 | 天津市、河南省指南 |

这五项制度不需要写得像法律条文一样复杂，但一定要**说清楚"谁在什么时候做什么事"**。好的制度文件，新人看完就知道自己要干什么。

#### 3.1.3 分层培训

不同层级的人员，需要了解的内容是不同的。综合国务院安委办文件和各省指南，培训对象通常分为四个层次：

| 层次 | 对象 | 培训重点 | 参考依据 |
|:-----|:-----|:---------|:---------|
| 决策层 | 企业领导、项目班子成员 | 政策法规要求、体系建设思路、资源保障 | 天津市、山东省指南 |
| 管理层 | 部门负责人、技术/安全管理人员 | 风险评估方法、管控措施设计、清单编制 | 天津市、河南省指南 |
| 执行层 | 班组长、分包负责人 | 隐患排查方法、措施落实要求 | 黑龙江省特色要求 |
| 操作层 | 一线作业人员 | 本岗位风险认知、个体防护、应急响应 | 新《安全生产法》、各省指南 |

一个常见的误区是：让所有人在同一堂课上学习同样的内容。结果是领导觉得太细、工人觉得太玄，谁也学不进去。分层培训可以提高效率，也更容易让培训内容与实际工作挂钩。

#### 【案例：土方工程项目组织搭建】

**案例简介：**

| 项目信息 | 说明 |
|:---------|:-----|
| 工程类型 | 基础工程-土方开挖 |
| 施工内容 | 基坑土方机械开挖、人工清底修坡、土方运输 |
| 基坑规模 | 开挖深度5.2m，属危险性较大的分部分项工程 |
| 地质条件 | 粉质粘土，需考虑不良地质风险 |
| 周边环境 | 距既有建筑8m，周边道路有车辆通行 |
| 主要设备 | 挖掘机、土方运输车辆 |

某基础工程土方项目开工前，项目经理组织成立了双控工作小组，成员包括安全总监、技术负责人、施工员、机械管理员以及土方分包班组长。明确了每个人的任务分工：技术负责人负责危险源辨识和风险评价，安全总监负责清单编制和审核，施工员负责措施现场落实，班组长负责班组级培训和日常巡查。

项目部依据企业级制度，编制了本项目《安全风险分级管控实施方案》和《隐患排查治理工作计划》，明确本项目风险分析的范围、方法、时间节点和责任人。开工前组织了三场分层培训：对项目管理人员培训风险评估方法和清单编制要求，对机械操作人员培训操作规程和岗位风险告知，对清底修坡工人培训个人防护和应急逃生。

### 3.2 第二步：确定风险点/划分辨识单元

组织机构搭建好了，现在开始正式进入分析工作。第一步是：**确定我们要分析什么。**

这一步的核心任务是**划定分析边界**——在正式开始风险分析之前，先明确我们要分析什么、分析到什么程度。

具体来说，需要做好三件事：
1. **明确工程范围**，获取分部分项清单，界定本次分析的施工内容
2. **确定边界条件**，包括地质条件、周边环境、施工方案等影响风险的因素
3. **系统识别风险点**，确保覆盖完整、没有遗漏

#### 3.2.1 什么是风险点

风险点是伴随风险的部位、设施、场所和区域，以及在这些场所实施的伴随风险的作业活动，或以上两者的组合。简单说，风险点就是"我们把风险贴在哪个标签上"——它是一线管理的基本单元。

风险点的划分需要把握五个原则：大小适中、便于分类、功能独立、易于管理、范围清晰。

**风险点划分依据**

风险点的划分应有明确的依据，确保系统性和完整性。主要划分依据包括：

1. **分部分项工程划分**：按照《建筑工程施工质量验收统一标准》（GB 50300）的分部分项工程划分原则，结合工程实际进行风险点归类。例如：地基与基础工程、主体结构工程、建筑装饰装修工程、建筑屋面工程、建筑给排水及供暖工程、通风与空调工程、建筑电气工程、智能建筑工程、建筑节能工程、电梯工程等分部工程，可进一步细分为若干风险辨识单元。

2. **施工工艺流程**：按照施工先后顺序和工艺特点，将连续施工过程划分为若干相对独立的作业单元，便于识别各工序的风险特征。

3. **空间位置分布**：根据施工现场平面布置和竖向分布，按区域、楼层、标段等空间维度划分风险点，便于现场管理和责任落实。

4. **设备设施类别**：针对大型机械设备、临时设施、特种设备等，按设备类型和功能系统划分风险点。

5. **作业活动性质**：按照作业活动的危险特性，如高处作业、动火作业、受限空间作业、起重吊装作业等进行分类划分。

#### 3.2.2 三维扫描法

实践中，推荐采用"三维扫描法"来系统识别风险点：

| 扫描维度 | 扫描对象 | 说明 |
|:---------|:---------|:-----|
| 部位/区域 | 具有潜在风险的物理位置或空间 | 基坑周边、堆土区、运输通道 |
| 设施/设备 | 可能引发风险的设备、设施或工具 | 挖掘机、运输车辆、临时用电 |
| 作业活动 | 伴随风险的作业过程 | 土方开挖、人工清底、土方运输 |

推荐用至少两种方法做交叉验证。例如，先用**区域划分法**（按施工平面图划分区域，逐一扫描），再用**工序分解法**（按工艺流程分解作业，逐一识别），把两者的结果合并去重，覆盖度最高。

#### 【案例：土方工程风险点识别】

土方工程的场景边界条件确定如下：

> 基坑开挖深度5.2m（属危大工程），土质为粉质粘土，周边距既有建筑8m。采用机械开挖为主、人工清底修坡为辅的施工方式。主要施工工序：施工准备→测量放线→分层机械开挖→人工清底修坡→土方外运→基底验槽。

采用三维扫描法识别出5个风险点：

```
土方工程风险点识别结果
  │
  ├── 作业活动扫描
  │     ├── RP-001 基坑土方机械开挖作业
  │     └── RP-003 人工清底修坡作业
  │
  ├── 设备设施扫描
  │     ├── RP-002 挖掘机作业
  │     └── RP-004 土方运输车辆作业
  │
  └── 部位区域扫描
        └── RP-005 基坑边土方堆放区
```

验证方法：列出土方工程常见事故类型——坍塌、车辆伤害、机械伤害、高处坠落——检查每个事故类型是否可以追溯到某个风险点。结果显示五个风险点覆盖了所有可能的事故类型，辨识完整。

### 3.3 第三步：危险源辨识

有了风险点，接下来要回答一个更本质的问题：**风险点的"根"是什么？** 答案是危险源——那些承载着能量的物体或物质。

#### 3.3.1 危险源的本质

**危险源是具有潜在能量的物体或物质，是风险的起点。** 这个定义的核心是"能量"——没有能量，就没有危险源。

在实际辨识中，有一个常见的误区：把"违章操作""管理缺陷"也当成危险源。这些不是危险源，而是导致危险源失控的原因（属于第二类危险源，后面的"威胁"会详细讨论）。危险源只有那几样承载着能量的"物体"。

#### 3.3.2 能量源扫描法（四问法）

扫描危险源，可以用"四问法"快速锁定：

| 问题 | 扫描目标 |
|:-----|:---------|
| 什么有质量？ | 物体、材料、土体、设备 |
| 什么有势能或动能？ | 高处物体、移动机械 |
| 什么有压力？ | 侧向土压力、水压力、压力容器 |
| 什么有化学能？ | 危险化学品、燃气、氧气乙炔 |

#### 3.3.3 辨识方法的选择

对于**作业活动类**风险点（如土方开挖），将工序分解到操作步骤，然后分析每个步骤中的危险源。

对于**设备设施类**风险点（如挖掘机），依据JGJ 59《建筑施工安全检查标准》等规范，列出设备的安全检查项目，再从中提炼危险源。

#### 【案例：土方工程危险源辨识】

对土方工程五个风险点进行危险源扫描，识别出4个危险源：

```
土方工程危险源扫描结果
  │
  ├── RP-001 基坑土方机械开挖作业
  │     ├── H001 基坑边坡土体（势能）——土体具有重力势能，边坡失稳时释放
  │     └── H002 挖掘机（动能）——机械运动部件具有动能，失控碰撞碾压
  │
  ├── RP-002 挖掘机作业
  │     └── H002 挖掘机（动能）——机械运动部件具有动能，失控碰撞碾压
  │
  ├── RP-003 人工清底修坡作业
  │     └── H001 基坑边坡土体（势能）——土体具有重力势能，边坡失稳时释放
  │
  ├── RP-004 土方运输车辆作业
  │     └── H003 运输车辆（动能）——车辆行驶具有动能，失控造成交通事故
  │
  └── RP-005 基坑边土方堆放区
        ├── H001 基坑边坡土体（势能）——土体具有重力势能，边坡失稳时释放
        └── H004 堆放土方（势能）——堆土具有重力势能，滑塌时冲击下方
```

这种分析方式的背后有一个**关键区分**："坡度过陡""堆载超限""违章操作"这些不是危险源，它们是后续要分析的**威胁**。危险源只有这几样承载能量的"物体"。

这里需要做一个关键区分：
- **危险源** = 第一类危险源（能量源）
- **威胁** = 第二类危险源（约束失效因素）

这个区分，恰恰是传统危险源辨识中经常理不清的地方。一趟分析下来，能量源和触发因素混在一起，分不清哪个是"根"哪个是"因"。用Bow-Tie的方法，这个逻辑就理顺了。

### 3.4 第四步：风险评价与分级

危险源识别出来后，接下来要从三个方面展开分析：**什么因素会触发危险源失控？失控后会发生什么？后果有多严重？**

这一步是整个风险分析的核心环节。我们需要依次识别出四个关键要素：初始事件、威胁、后果，然后通过LS法对后果进行风险评估。

#### 3.4.1 初始事件识别——风险分析的"分水岭"

**初始事件**是风险分析的中心节点。它描述的是危险源"失控"的中性状态，不是灾难后果。

**三条判断原则**（必须全部通过）：

| 原则 | 检查问题 |
|:-----|:---------|
| 中性描述 | 是否描述"失控状态"而非"灾难后果"？ |
| 单一事件 | 是否聚焦一个明确的失控节点？ |
| 可识别性 | 能否通过工程或管理措施进行控制？ |

一个简单的检验方法："人员被埋"不是初始事件（这是后果），"基坑边坡坍塌"才是初始事件（这是中性描述）。

#### 【案例：土方工程初始事件】

```
土方工程初始事件识别结果
  │
  ├── H001 基坑边坡土体
  │     ├── IE001 基坑边坡坍塌——土体支撑力不足导致整体失稳
  │     └── IE005 边坡局部坍塌/落石——边坡局部失稳导致土体脱落
  │
  ├── H002 挖掘机
  │     ├── IE002 机械碰撞/碾压人员——机械运动轨迹侵入人员所在区域
  │     ├── IE003 机械失控——操作机构失灵导致非预期运动
  │     └── IE004 机械倾覆——机械重心偏移超出稳定范围
  │
  ├── H003 运输车辆
  │     ├── IE006 车辆碰撞/碾压人员——车辆行驶轨迹侵入人员所在区域
  │     └── IE007 车辆失控——制动转向系统失灵
  │
  └── H004 堆放土方
        └── IE008 堆土滑塌/边坡超载坍塌——堆土自稳性不足引发滑移
```

共识别8个初始事件，每个初始事件都对应到特定的危险源。这些初始事件构成了风险分析的"结"。

#### 3.4.2 威胁识别——直接触发因素

有了初始事件，接下来问：**什么样的事会让危险源失控？** 这些"直接触发因素"就是威胁。

识别威胁，同样可以用"三维扫描法"，从三个维度展开：

| 维度 | 扫描对象 | 典型威胁示例 |
|:-----|:---------|:-------------|
| 人（行为） | 不安全行为 | 违章操作、超挖、疲劳作业 |
| 物（状态） | 设备、材料异常 | 坡度过陡、堆载超限、设备故障 |
| 环（环境） | 环境条件突变 | 降雨侵蚀、不良地质、道路湿滑 |

**为什么不从"管理因素"识别威胁？**

有人可能会问：传统的事故致因分析通常有"人、物、环、管"四个维度，为什么这里只有"人、物、环"三个维度，把"管理"去掉了？

这不是疏忽，而是有意为之。"管理"不是威胁，而是**控制措施的载体**。威胁是那些**直接导致危险源失控的因素**——坡度过陡、降雨侵蚀、违章操作，这些都是可以直接观察到的现象。而"管理制度不健全""安全培训不到位"这些管理因素，本身不会直接触发事故，它们是通过导致"坡度过陡没人管""违章操作没人查"才间接发挥作用的。

换句话说：
- **威胁** = 直接导致初始事件的因素（人、物、环）
- **管理缺陷** = 控制措施失效的原因，属于"措施为什么没起作用"的范畴

如果把管理因素也列为威胁，会导致两个后果：一是混淆了"直接原因"和"间接原因"的界限，二是让后续的控制措施设计失去针对性——你很难针对"管理制度不健全"设计一个具体的控制措施，但针对"坡度过陡"可以设计"坡度复核验收"这个具体措施。

因此，威胁识别聚焦在"人、物、环"三个可以直接观察的维度，管理因素则留到控制措施设计和保障活动环节去处理。

#### 【案例：以H001（基坑边坡土体）为例】

```
危险源：基坑边坡土体（H001）
  │
  ├── T001 坡度过陡（物）
  │
  ├── T002 堆载超限（物）
  │
  ├── T003 降雨侵蚀（环）
  │
  ├── T004 机械振动（环）
  │
  ├── T005 不良地质（环）
  │
  ├── T006 超挖（人）
  │
  └── T012 支护失效（物）
         │
         ▼
  初始事件：基坑边坡坍塌（IE001/IE005）
```

从"人-物-环"三个维度扫描，一个危险源（H001）就识别出了7个威胁。这些威胁各对应不同的触发路径，后续的控制措施需要分别针对每个威胁来设计。

**威胁闭包验证**：参照GB 6441-2025，土方工程相关的事故类型有坍塌、车辆致害、机械致害、高处坠落。检查所有识别出的威胁，看它们是否覆盖这些事故类型的触发路径。如果某个事故类型（比如高处坠落）没有对应的威胁，说明需要补充。这种验证方法保证了辨识的完整性。

#### 3.4.3 后果识别

后果是初始事件发生后导致的最终损失，是风险的终端输出。

识别后果时，可以从四个维度评估：

| 维度 | 评估内容 | 典型示例 |
|:-----|:---------|:---------|
| **人员伤害** | 人员伤亡的严重程度 | 死亡、重伤、轻伤、职业病 |
| **财产损失** | 设备设施的损毁程度 | 设备损坏、材料报废、建构筑物损毁 |
| **环境影响** | 对周边环境的影响 | 土壤污染、水体污染、大气污染 |
| **声誉影响** | 对企业和社会形象的影响 | 负面舆情、行政处罚、法律责任 |

判断一个结果是否属于后果，可以用三个问题检验：
- 这个损失是最终结果还是中间过程？（必须是最终结果）
- 这个损失能直接归因于初始事件吗？（必须有直接因果关系）
- 这个损失是合理的可预见后果吗？（必须是可预见的）

#### 【案例：土方工程后果】

```
土方工程后果识别结果
  │
  ├── IE001 基坑边坡坍塌（坍塌）
  │     ├── C001 人员被埋伤亡
  │     ├── C002 设备损毁
  │     └── C005 工期延误
  │
  ├── IE002 机械碰撞/碾压人员（车辆致害）
  │     └── C003 人员伤亡
  │
  └── IE004 机械倾覆（机械致害）
        └── C004 设备损毁
```

后果的评估主体涉及人员伤亡、财产损失和工期影响三个维度。其中人员伤亡是风险评价的核心关注点。

#### 3.4.4 LS风险评估——确定风险等级

风险评估采用LS法（R = L × S），其中L是事故发生的可能性，S是后果的严重程度。

**风险等级划分标准：**

| 风险等级 | 颜色 | R值区间 | 说明 |
|:--------:|:----:|:--------:|:-----|
| 高风险 | 红 | 15～25 | 极易引发群死群伤 |
| 中风险 | 橙 | 10～12 | 可能引发人员重伤 |
| 低风险 | 黄 | 5～8 | 可能导致人员轻伤 |
| 很低风险 | 蓝 | 1～4 | 轻伤及以下 |

#### 【案例：土方工程风险评价结果】

|:-----|:-------|:---------|:--:|:--:|:--:|:---------|:---------|
| 人员被埋伤亡 | H001 | IE001 | 3 | 5 | **15** | **高风险** | **企业级** |
| 人员伤亡 | H002 | IE002 | 3 | 5 | **15** | **高风险** | **企业级** |
| 设备损毁 | H002 | IE004 | 3 | 4 | **12** | **中风险** | **项目级** |
| 工期延误 | H001 | IE001 | 3 | 3 | 9 | **中风险** | **项目级** |

### 3.5 第五步：制定管控措施

前一阶段回答了"有哪些风险"和"风险有多大"，现在进入最关键的环节——**如何管住这些风险？**

这一步的核心是设计**控制措施**（阻止威胁触发初始事件）和**应急措施**（减轻后果损失）。

#### 3.5.1 控制措施——阻止威胁触发初始事件

**控制措施是在初始事件发生前阻止威胁的屏障。** 每个控制措施都针对特定的威胁进行设计，放在风险分析图上就是左侧的防护线。

**五层级措施优先级：**

管控措施的设计不是"想到什么写什么"，而是按照以下优先级从高到低选择：

| 优先级 | 层级 | 编码 | 适用场景 | 示例 |
|:------:|:-----|:----:|:---------|:-----|
| 1（最高） | 消除 | E | 能从源头消除危险源 | 取消高危作业、改变施工工艺 |
| 2 | 替代 | S | 无法消除时用低风险方案 | 机械替代人工、预制替代现浇 |
| 3 | 工程控制 | EC | 核心防护手段 | 物理隔离、限位装置、监测系统 |
| 4 | 管理控制 | MC | 制度流程保障 | 操作规程、审批程序、技术交底 |
| 5（最低） | 个体防护 | PPE | 最后一道防线 | 安全帽、安全带、防护手套 |

**纵深防御规则：**

对于高风险路径，必须设置多层、多类型的屏障，不能只是"写制度"了事。具体规则如下：

- 高风险路径：必须有工程控制（EC）+ 管理控制（MC）+ 个体防护（PPE）
- 中风险路径：必须有管理控制（MC）+ 个体防护（PPE），建议增加工程控制
- 同一威胁路径上，至少设置多个不同层级的控制措施

这条规则的核心意思是：**不能只靠"写制度"来管住高风险。** 对于基坑边坡坍塌这种可能导致群死群伤的风险，必须有物理防护措施——比如放坡、位移监测这些实实在在的工程手段。

#### 【案例：土方工程控制措施设计——以H001为例】

```
危险源：基坑边坡土体（H001）
  │
  ├── T001 坡度过陡
  │     ├── CM001 按1:0.5放坡（EC）
  │     └── CM002 坡度复核验收（MC）
  │
  ├── T002 堆载超限
  │     ├── CM003 堆土距离≥3m限载（MC）
  │     └── CM004 设置堆土限位标识线（EC）
  │
  ├── T003 降雨侵蚀
  │     ├── CM005 设置排水沟和集水坑（EC）
  │     └── CM006 暴雨天气停止坑边作业（MC）
  │
  ├── T004 机械振动
  │     └── CM007 挖掘机距坑边≥1m限位（EC）
  │
  ├── T005 不良地质
  │     └── CM008 边坡位移监测及预警（EC）
  │
  ├── T006 超挖
  │     └── CM009 基底标高复核验收（MC）
  │
  └── T012 支护失效
        ├── CM010 支护结构设计方案专家论证（EC）
        └── CM011 支护施工过程质量验收（MC）
```

8个威胁共设计11道控制措施。H001关联的"人员被埋伤亡"是高风险（R=15），按照纵深防御规则检查：每个威胁路径上都设置了工程控制（EC）或管理控制（MC）措施，部分高风险路径同时设置了两类措施，满足纵深防御要求。

#### 3.5.2 应急措施——减轻后果严重程度

**应急措施是在初始事件发生后减轻后果的屏障。** 放在风险分析图上，就是右侧的缓解线。

控制措施和应急措施的分工很明确：
- 控制措施的目的是**不让事情发生**
- 应急措施的目的是**事情发生了，把损失降到最低**

**应急措施的三层级：**

| 层级 | 响应时机 | 说明 | 典型示例 |
|:-----|:---------|:-----|:---------|
| **工程控制** | 事发时自动/即时响应 | 优先选择 | 逃生通道、泄压装置、紧急切断阀、急救箱、消防器材 |
| **管理控制** | 事发后人工响应 | 补充选择 | 应急预案、疏散路线、应急广播 |
| **个人防护** | 减轻伤害程度 | 作用有限 | 安全帽、安全带、防护眼镜、绝缘手套 |

**设计约束：**

- 对高后果风险（如可能导致群死群伤），必须设置**独立多层**应急措施
- 禁止仅设管理型应急措施而不设工程型应急措施——比如不能只写"应急预案"，必须有"逃生通道"这种物理设施

#### 【案例：土方工程应急措施设计——以边坡坍塌为例】

```
                        初始事件：基坑边坡坍塌（IE001）
                              │
                              ▼
              ├── EM001 沿基坑每20m设逃生通道，间距≤20m（EC）
              ├── EM002 配备应急救援物资（EC）
              ├── EM003 编制坍塌专项应急预案并定期演练（MC）
              └── EM004 与周边医疗机构建立应急联动（MC）
                              │
                              ▼
                        后果：人员被埋伤亡（C001）
```

#### 3.5.3 从分析成果到管控清单

前面四步（风险点→危险源→初始事件→后果）加上第五步的控制措施和应急措施设计，已经完成了风险分析的全部内容。现在需要把这些分析成果转化为一份**可执行、可检查、可追溯**的管控清单。

**清单的核心作用：**

| 作用 | 说明 |
|:-----|:-----|
| **现场指导** | 告诉作业人员这个工序有什么风险、怎么防范 |
| **管理依据** | 为安全检查、隐患排查提供明确的检查项 |
| **责任追溯** | 明确每项措施的管控层级（企业级、项目级、班组级、作业人员级） |
| **动态更新** | 随工程进度变化及时调整管控重点 |

**清单从哪里来？**

清单不是凭空编制的，而是前面分析成果的"汇总表"：

```
风险点（RP-001）
  → 危险源（H001基坑边坡土体）
    → 威胁（T001坡度过陡、T002堆载超限...）
      → 控制措施（CM001放坡、CM003限距...）
    → 初始事件（IE001基坑边坡坍塌）
      → 后果（C001人员被埋）
        → 应急措施（EM001逃生通道、EM002应急物资...）
```

把这些要素按行展开，就是管控清单的一条记录。

**清单的关键字段：**

一份实用的管控清单至少应包含以下信息：

| 信息类别 | 核心字段 | 作用 |
|:---------|:---------|:-----|
| **场景信息** | 分部工程、分项工程、工序 | 明确这条记录对应哪个施工环节 |
| **风险信息** | 危险源、威胁、初始事件、后果 | 说明风险是什么、怎么来的 |
| **评估信息** | L、S、R、风险等级 | 判断风险大小，确定管控优先级 |
| **措施信息** | 控制措施、应急措施 | 明确怎么防范、怎么应急 |
| **责任信息** | 管控层级 | 明确谁来负责 |

清单编制完成后，风险分级管控的"分析阶段"告一段落。

---

### 3.6 第六步：高及中风险控制措施评估

对所有风险完成评估和管控措施设计之后，还需要对**高风险**和**中风险**做一次专项评估——确认这些措施的可行性、安全性和可靠性。

#### 3.6.1 评估要求

施工企业、项目部应当在重大风险及较大风险控制措施实施前，或在其可行性、安全性及可靠性出现偏差时，由企业主要负责人或项目负责人组织相关人员，对风险控制措施进行评估。

评估内容：
- **可行性**：措施在技术、经济、工期上是否可行？
- **安全性**：措施实施后是否有次生风险？
- **可靠性**：措施能否持续有效，不会因为日常磨损或管理松懈而失效？

---

### 3.7 第七步：风险告知与分级管控

最后一步，是把前面所有分析成果转化为现场的实际行动。这一步包含三个核心任务：**风险告知、分级管控、措施验证。**

#### 3.7.1 风险告知——让每个人都清楚风险在哪里

风险分析做完了，如果信息只停留在管理人员的文件里，等于没有做。风险告知的目的是让每一个进入施工现场的人——从项目经理到一线作业人员——都知道自己面临什么风险、应该如何应对。

施工现场常见的告知方式包括：

| 告知方式 | 适用对象 | 核心要求 |
|:---------|:---------|:---------|
| **安全生产风险公示牌** | 高、中级风险 | 在施工现场显著位置公示风险点、风险等级、管控措施、应急措施、责任人 |
| **岗位风险告知卡** | 每个工种/岗位 | 标明主要风险、事故类别、管控措施、应急措施、报告方式 |
| **安全警示标志** | 特定风险区域 | 按GB 2894设置禁止、警告、指令、提示标志 |
| **安全技术交底** | 作业前人员 | 分部分项工程开工前，对作业人员进行详细风险告知和措施交底 |
| **三级安全教育** | 新进场人员 | 企业、项目、班组三级，包含风险告知内容 |
| **班前会** | 每日作业人员 | 每日作业前讲解当日风险点和防范措施 |
| **公告栏/宣传栏** | 全体人员 | 定期更新风险信息、事故案例、管控动态 |
| **二维码/信息平台** | 全体人员 | 扫码查看风险详情、动态更新、实时预警 |

**各省特色要求补充：**

- **天津市**（2025）：强调"一风险一档案"，要求重大风险建立专项档案，包含风险辨识、评估、管控全过程记录
- **河南省**（2019）：要求风险告知卡必须包含"四色分布图"，直观展示作业区域风险分布
- **广东省**（2024）：强调建设单位牵头组织首次风险告知，要求在开工作业前完成全面风险交底
- **黑龙江省**（2019）：要求高风险作业必须执行"班前风险确认"制度，作业前由班组长逐条确认风险管控措施落实

#### 3.7.2 分级管控——责任落实到对应层级

分级管控的核心原则是：**风险等级越高，管控层级越高。**

| 风险等级 | 颜色 | 管控层级 | 责任人 | 土方案例应用 |
|:--------:|:----:|:---------|:-------|:-------------|
| 高风险（一级） | 红 | 企业级 | 企业主要负责人 | 基坑边坡坍塌 → 企业挂牌督办 |
| 中风险（二级） | 橙 | 项目部级 | 项目经理 | 机械倾覆 → 项目主责 |
| 低风险（三级） | 黄 | 班组级 | 班组长 | 人工清底 → 班组落实 |
| 很低风险（四级） | 蓝 | 岗位级 | 作业人员 | 常规作业 → 员工自查 |

需要注意一个管理细节：**上一级管控的风险，下一级必须落实管控措施。** 企业级管控的基坑边坡坍塌风险，项目经理同样要抓落实，班组长同样要在现场执行放坡和监测要求。分级管控不是"分而不管"，而是明确主责，各级都在链条上。

**动态调整机制**：风险等级不是一成不变的。随着施工进度推进、环境条件变化、工艺方法调整，风险等级可能升高或降低。有效管控的可申请降级，应控未控的应提级管控。

#### 3.7.3 关键保障活动——验证措施是否真的有效

措施制定了，风险告知了，责任也分了——**但怎么知道这些措施真的在起作用？**

这里需要引入**关键保障活动**的概念。关键保障活动是验证控制措施或应急措施是否持续有效的活动，回答"怎么知道措施还好使？"。

保障活动可以分为四类：

| 类别 | 验证问题 | 案例 | 建议频次 | 责任人 |
|:-----|:---------|:-----|:--------:|:-------|
| A.检查维修 | 装好了没？还好着吗？ | 放坡坡度检查、排水沟是否通畅 | 每班次 | 安全员/施工员 |
| B.教育培训 | 用的人会吗？ | 应急预案演练、操作规程考核 | 每季度 | 项目安全总监 |
| C.验证测试 | 设计对吗？功能靠谱吗？ | 基坑支护方案专家论证 | 开工前/变更时 | 技术负责人 |
| D.独立审计 | 自检结果是真的吗？ | 公司级专项检查、第三方审核 | 每月/每季度 | 企业安全部门 |

每一个控制措施和应急措施，都应该至少有对应的保障活动。没有保障活动的措施，理论上无法验证其有效性。

#### 3.7.4 从风险管控清单到隐患排查清单

关键保障活动和隐患排查是什么关系？答案是：**保障活动就是隐患排查的来源。**

国务院安委办〔2016〕11号文件明确要求隐患排查清单来源于管控清单。具体转化逻辑如下：

```
风险管控清单中的一条措施：
  CM003 严格限制堆土距离≥3m（管理控制措施）
  ↓ 对应的关键保障活动：
  CA002 检查堆土距坑边距离是否≥3m（检查维修类，每班次）
  ↓ 纳入隐患排查清单后：
  排查项：堆土距基坑边的距离是否≥3m
  排查方法：现场测量
  排查人：安全员
  排查频次：每班次
  隐患分类：生产现场类隐患
  隐患级别：一般隐患
```

这个转化逻辑确保了"两张清单"不是各自独立的——隐患排查清单的每一条，都对应着风控清单中某条措施的保障活动。反过来，隐患排查中的每一个"不合格"项，也都可以追溯到某条管控措施失效。

## 第四章 各方职责分工

风险分级管控不是施工企业一家的事。国务院安委办和多省文件都明确了参建各方在双控中的职责分工。

| 责任方 | 责任定位 | 核心职责 |
|:-------|:---------|:---------|
| **建设单位** | 首要责任 | 开工前牵头组织安全风险识别，足额支付安全费用，督促各参建方落实双控 |
| **施工单位** | 主体责任 | 总包统一协调各分包单位，建立体系制度，编制清单并执行 |
| **监理单位** | 监理责任 | 审查风险管控资料，现场巡查措施落实情况，发现重大隐患责令停工 |
| **勘察设计单位** | 源头责任 | 在勘察报告和设计文件中注明安全风险重点部位，列明危大工程清单 |

一个项目双控机制能否真正落地，很大程度上取决于建设单位是否真正把"首要责任"承担起来（比如牵头组织第一次风险识别会议），以及总包单位是否有能力协调各分包单位统一执行统一的管控标准。

## 第五章 持续改进机制

风险分级管控不是"一次性"的工作。随着施工推进，风险在变化，措施的有效性也在变化。建立一个持续改进的机制，是风险管控能够长期有效运转的前提。

### 5.1 PDCA循环

P（策划）→ D（实施）→ C（检查）→ A（改进），这个经典的管理循环同样适用于风险分级管控：

```
P：建立组织、制度，确定风险点，辨识危险源，评价风险，制定措施
D：执行管控措施，开展风险告知和培训
C：验证措施有效性（关键保障活动），隐患排查
A：评估效果，发现偏差，调整措施和清单
```

PDCA循环之所以有效，深层原因在于它顺应了一个基本规律：**变化是常态，不变才是例外。** 一份年初编制的清单到了年中可能已经与现场实际脱节——不是清单有问题，而是风险本身在变化。PDCA中的"检查"和"改进"两个环节，正是对"变化"这个事实的制度化回应：定期评审是为了主动发现偏差，动态更新是为了及时校正方向。从这个角度看，持续改进机制的本质，就是用"有意识的管理节奏"来应对"不可预测的外部变化"。

### 5.2 动态更新触发条件

广东省2024年文件明确了以下几种情况应当更新：
- 法律法规、标准规范发生变化
- 周边环境发生变化
- 施工工艺或工程措施发生变化
- 工程进度推进导致风险变化
- 发生事故或险情后

#### 【案例：土方工程动态更新】

```
基坑开挖至基底标高前 →
基坑深度变化导致边坡稳定性风险变化 →
重新评估危险源和威胁 →
更新风险分级管控清单 →
更新隐患排查清单 →
进入新的管控周期
```

### 5.3 定期评审

天津市指南要求对双重预防机制运行情况进行定期评审，评审内容包括：
- 制度的适宜性、充分性和有效性
- 风险管控的实际效果
- 隐患排查治理的闭合率

### 5.4 信息化

随着越来越多的项目采用信息化管理手段，风险分级管控的数字化也在推进中。从纸质清单到电子表格再到管理平台，数据互联互通、异常自动预警，正在从趋势变成现实。

## 第六章 总结与建议

### 6.1 九大关键成功要素

综合全国二十余份文件的研究和多个项目的实践经验，风险分级管控能否真正落地，取决于以下九个要素：

**1. 领导重视是前提。** 双控不是安全部门一个部门的事。主要负责人不亲自抓，这个机制大概率做不起来。

**2. 全员参与是基础。** 从企业负责人到一线作业人员，都要纳入体系。不是所有人都要参与风险分析，但所有人都有自己要承担的角色。

**3. "两张清单"是核心工具。** 风控清单和排查清单是整个机制最核心的两份文件。关键在于"排查清单来源于管控清单"这一联动逻辑必须贯穿始终。

**4. 动态更新是生命力。** 风险随施工进度变化。一份清单编完锁在柜子里用一年，等于没有做。

**5. 培训教育要分层。** 决策层、管理层、执行层、操作层各有不同的培训内容，不应该放在同一堂课里。

**6. 重大风险要"加码"管理。** 事前评估、提级管控、高频报告——对可能导致群死群伤的风险，怎么重视都不过分。

**7. 闭环管理不能留缺口。** 排查→登记→整改→验收→销号，少一环都不叫闭环。

**8. 信息化是趋势。** 从纸质到数字，从静态到动态，异常预警和数据分析将极大提升管控效率。

**9. 持续改进是"最后一公里"。** 没有持续改进的机制，是做不长的。PDCA要贯穿始终。

### 6.2 引入Bow-Tie的核心价值

在前面的七步流程中，我们已经看到了这种基于因果链的风险分析方法在每个环节中的具体应用。总结起来，引入这套方法的核心价值体现在五个方面：

| 维度 | 传统做法的局限 | 引入因果链分析方法后的提升 |
|:-----|:--------------|:-------------------|
| **可视化** | 清单和表格，难以直观理解风险链条 | 因果树状图，一目了然 |
| **结构化** | 辨识依赖个人经验，容易遗漏 | 威胁闭包验证，系统化覆盖 |
| **措施设计** | 容易"一刀切"——笼统描述 | 五层级纵深防御，高风险必须有工程控制 |
| **可验证** | 措施写在纸上，有没有用不知道 | 每个措施有对应的关键保障活动 |
| **与隐患排查联动** | 容易"两张皮" | 保障活动直接映射为排查项目，逻辑统一 |

这套方法不是用来取代双控的，而是用来让双控的每个环节做得更扎实。它的最大价值是：**让风险管理从"交作业"变成"真在用"。**

如果把风险管控仅仅理解为"一套流程"或"一份清单"，它的价值是有限的。真正让这套体系发挥作用的，是一种渗透到组织日常中的思维方式——**在任何时候都保持对潜在失衡的警觉，在一切看似正常的状态下主动寻找可能失控的节点。** 最高明的风险管理不是在事故发生后才匆忙补救，而是在一切看起来没有问题的时候，管理者已经在问："如果出事，会从哪里开始？" 这种思维方式的养成，比任何一张清单都更能决定一个项目的安全水平——因为清单可以复制，而思维方式才是真正不可替代的。

### 6.3 结语

回到最开头那个问题：**风险分级管控到底怎么干？**

我们的建议是三步：**搭好架子、想透风险、持续改进。** 搭架子是第一步（组织机构、制度、培训），想透风险是核心（七步流程加因果链分析），持续改进是长效机制。

一个项目做得好不好，不看他的体系文件有多厚，而看三个问题：
1. 他的风险清单和施工现场的真实风险对得上吗？
2. 他的隐患排查清单是从风险管控清单转化过来的吗？
3. 他的班组长和作业人员知道自己岗位的风险是什么吗？

这三个问题都答"是"，双控就是真的在落地。这也是本文想传递的核心信息。

> **"把风险控制在隐患形成之前，把隐患消灭在事故前面。"**
>
> ——国务院安委办〔2016〕11号
